# Getting Support

Thank you for using our projects! Here's how to get help.

## Documentation

Start by reviewing the project's README and any available documentation in the repository.

## Questions and Issues

### General Questions

For general questions about using the project:
- Check existing [GitHub Issues](../../issues) to see if your question has been answered
- Open a new issue with the "question" label if needed

### Bug Reports

If you've found a bug:
- Search existing issues first to avoid duplicates
- Open a new issue with a clear description and steps to reproduce
- Include relevant system information and error messages

### Feature Requests

For new feature ideas:
- Check if a similar request already exists
- Open an issue describing the feature and its use case

## Response Times

We monitor issues regularly but cannot guarantee specific response times. Issues will be prioritized based on severity and impact.

## Security Issues

**Do not** open public issues for security vulnerabilities. See our [Security Policy](SECURITY.md) for reporting instructions.
